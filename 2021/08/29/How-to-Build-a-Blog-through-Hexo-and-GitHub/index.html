通过Hexo+GitHub搭建个人博客

Posted on 2021-08-29 Edited on 2021-08-31 In technology

环境

  • Windows10 64bit

准备工作

安装Node.js

  • 查看是否已安装Node.js
    打开cmd，输入node -v，如果显示Node版本号，说明已安装对应版本Node.js

  • 如果没有安装
    官网下载地址：https://nodejs.org/en/
    我选择的版本是：

    14.17.5 LTS Recommended For Most Users

    如果官网下载太慢，可试试网盘：https://pan.baidu.com/s/1Yw3gM33NwHY7JbDMKF4EUw
    提取码：i7tp

安装Git

  • 查看是否已安装Git
    cmd中输入git --version，如果返回Git版本号，说明已安装对应版本Git
  • 如果没有安装
    官网下载地址：https://git-scm.com/

为GitHub配置SSH密匙

参见博客：GitHub如何配置SSH Key

部署Hexo到Github

  • 安装扩展

  • 本地文件提交到 GitHub Pages

注：hexo g 和 hexo d 可以简写为 hexo g -d
在浏览器中输入 https://yourgithubid.github.io 就可以看到 Hexo 与 GitHub Pages 已经成功关联了。

新建博文

一些注意事项：

  • 如果文章标题有空格，需要用引号把整个标题引起来
  • 文章标题默认为一级标题，因此正文要从二级标题开始
  • 正文从—之后开始，—之间的内容为title、tag、category等
  • 文章写好之后通过

推送到GitHub Pages，有时候需要等几分钟网页才会更新改动

配置主题

  • 我最喜欢的主题 NexT
    • 在themes文件夹下打开gitbash，克隆主题
  • 更改_config.yml
  • 推送到GitHub Pages
  • 更多主题 hexo themes

常见问题和解决方法

  • 执行 hexo d 报错：

原因：没安装hexo-deployer-git插件
解决方法：在站点目录下安装插件

  • github.io 页面显示 master 分支下的内容而非 main 分支下的内容
    原因：_config.yml 中 Deployment 下的 branch 设置为了 master 而不是 main
