Solve Can't Find Shared Folder in Ubuntu VM

Posted on 2021-09-28

起因

Ubuntu虚拟机中设置与主机的共享文件夹后，在/mnt/hgfs目录下找不到该文件夹

环境：Windows 10 + VMware Workstation 16.0.0 + Ubuntu 18.04

本文主要解决的问题

  • /mnt/hgfs下找不到共享文件夹
  • 图标“安装 VMware Tools”为灰色
  • 设置中只能看到一个CD/DVD，找不到CD/DVD2和软盘
  • 解压VMwareTools的命令
  • 安装vmware-install.pl过程中的选择

解决步骤

一、设置共享文件夹

虚拟机 → 设置 → 选项 → 共享文件夹 → 总是启用 → 添加 → 选择主机路径 → 确定

假如此时能在 /mnt/hgfs/ 路径下找到设置的共享文件夹，则说明设置成功，无需再进行以下操作。

但根据我的经验，一般第一次会设置成功，但重启后就找不到了，所以如果重启后找不到设置的共享文件夹了，则需要重装VM tools，具体步骤如下：

二、解决无法安装 VMware Tools

虚拟机 → 安装 VMware Tools

如果发现 VMware Tools 是灰色的，则需进行以下操作：

即：将 CD/DVD (SATA)、CD/DVD 2 (SATA)和软盘都设置为“启动时连接”，如果找不到CD/DVD 2 (SATA)和软盘，则需要先自己添加。

重启虚拟机，此时就可以正常安装VMware Tools了

三、安装VMware Tools

  • 双击桌面上的DVD文件

  • 将VMwareTools-10.3.22-15902021.tar.gz拷贝到home目录下

  • 在home目录下打开terminal，进行以下操作

  • 在接下来的所有选择中，都选yes，安装成功会看到Enjoy

  • 重启客户机就可以在 /mnt/hgfs/下看到共享文件夹了

Posted on 2021-08-29 Edited on 2021-08-31 In technology

环境

  • Windows10 64bit

准备工作

安装Node.js

  • 查看是否已安装Node.js
    打开cmd，输入node -v，如果显示Node版本号，说明已安装对应版本Node.js

  • 如果没有安装
    官网下载地址：https://nodejs.org/en/
    我选择的版本是：

    14.17.5 LTS Recommended For Most Users

    如果官网下载太慢，可试试网盘：https://pan.baidu.com/s/1Yw3gM33NwHY7JbDMKF4EUw
    提取码：i7tp

安装Git

  • 查看是否已安装Git
    cmd中输入git --version，如果返回Git版本号，说明已安装对应版本Git
  • 如果没有安装
    官网下载地址：https://git-scm.com/

为GitHub配置SSH密匙

参见博客：GitHub如何配置SSH Key

常见问题和解决方法

  • 执行 hexo d 报错：

原因：没安装hexo-deployer-git插件
解决方法：在站点目录下安装插件

  • github.io 页面显示 master 分支下的内容而非 main 分支下的内容
    原因：_config.yml 中 Deployment 下的 branch 设置为了 master 而不是 main
